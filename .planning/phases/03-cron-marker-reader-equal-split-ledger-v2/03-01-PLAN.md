<objective>
**Phase Goal (from ROADMAP.md, verbatim):** When the cron runs against a session that has N markers since the previous ledger row, it emits N `revenium meter completion` calls whose per-field token splits sum exactly to the session delta, with marker-aware idempotency surviving any partial multi-call failure.

This single fat plan is locked by D-01. It ships ALL 15 Phase 3 requirements in one atomic migration because **Pitfall 8** makes partial adoption incorrect by construction: per-marker ledger lines + deterministic `--transaction-id` + per-call ledger writes MUST ship together or the load-bearing idempotency invariant breaks.

The task ordering below (D-02) is structured so each commit leaves the system in a working state — the backward-compat fallthrough (CRON-07) is the LAST invariant to break. Tasks T01-T03 are non-behavior-changing scaffolding; T04 is the cutover; T05-T08 complete the migration with fail-open tolerance, documentation, and end-to-end tests.

Purpose: Per-task attribution lands in Revenium analytics so spend is grouped by *what the agent was doing*, not just by session, with idempotency strong enough to survive partial multi-call failures.

Output: A `revenium-hermes.ledger` file that contains v2 5-field rows for marker-aware sessions and v1 4-field rows for legacy data (both readable forever); N Revenium API calls per session per tick (each carrying its own `--task-type`, `--operation-type`, and `--transaction-id`); a `cron.lock` file ensuring no overlapping ticks corrupt the ledger.
</objective>

Marker JSONL schema (from Phase 2 — already test-enforced by test_marker_file_schema):
```json
{"muid": "01893b8a300abcdef0123456789abcdef", "ts": 1715515200.0, "sid": "test-session", "task_type": "code_review", "operation_type": "GUARDRAIL"}
```
Required keys: `muid, ts, sid, task_type, operation_type`. Optional: `turn_seq, agent, trace_id, model`. `muid` regex: `^[0-9a-f]{33}$`. `task_type` regex: `^[a-z][a-z0-9_]{1,47}$`. `operation_type` enum from OpenInference span_kind. Line < 1024 bytes.

Ledger v1 format (read-only, never re-written per D-09):
```
HERMES:<sid>:<total_tokens>:<ts>
```

Ledger v2 format (D-07, all new writes use this — ONE muid per row, locked by B1):
```
HERMES:<sid>:<total_tokens>:<ts>:<muid>
```
Discrimination by `len(line.split(':'))`: 4 fields = v1, 5 fields = v2. Safe because Hermes session_ids are `YYYYMMDD_HHMMSS_<hex>` with no colons (verified via Hermes Agent docs). Field 5 is EXACTLY one muid (33-char hex, or the `unclassified-<ts_short>` synthetic placeholder for the zero-marker path). The prior-muids set for `(sid, total_tokens)` is the UNION of field 5 across ALL matching v2 rows.

Create a new standalone Python module per D-03, D-06. Stdlib-only (`from decimal import Decimal`). The module exposes TWO production helpers plus shared constants:

1. **`equal_split(delta: dict, n: int) -> list[dict]`** — returns N dicts whose per-field values sum byte-exact to the input. Integer fields use floor-division with remainder absorbed by the last marker; the `cost` field uses `Decimal` arithmetic quantized to 6 decimal places with the remainder absorbed by the last marker (per RESEARCH.md Pattern 5). Module docstring MUST record the plug-in shape for future strategies per D-06:

   ```
   def weighted_split(delta_fields, markers_with_length_hints) -> list[dict]
   def guardrail_estimator_split(delta_fields, markers, guardrail_share_estimator) -> list[dict]
   ```

2. **`parse_prior_state(ledger_path: str, sid: str, total_tokens: int) -> tuple[float, set[str]]`** — extracted per B6 revision so the SAME function is imported by both `hermes-report.sh`'s marker-reader heredoc (via `from split_strategies import parse_prior_state`) AND `test_ledger_v1_v2_discrimination`. Returns `(prior_ts, prior_muids)`:
   - Read `ledger_path` line-by-line; ignore lines not starting with `HERMES:{sid}:`.
   - For each matching line, split on `:`. Field-count discrimination:
     - 4 fields = v1: contributes its `ts` (field 4) to the candidate cutoff; contributes NO muids.
     - 5 fields = v2: contributes its `ts` (field 4) to the candidate cutoff; contributes field 5 (one muid) to `prior_muids` IFF `total_tokens` matches field 3 (otherwise the muid is for a different total_tokens — do not include).
   - **v2-takes-precedence cutoff (Pitfall D mitigation):** `prior_ts = max(ts across all v2 rows for this sid)`. Only fall back to `max(ts across v1 rows for this sid)` when no v2 row exists for this sid at all.
   - `prior_muids = union(field-5 across v2 rows where field-3 == total_tokens)`. (Muids from OTHER total_tokens values are NOT included — they belong to a different delta window.)
   - Returns `(0.0, set())` if no matching rows.
   - Defense-in-depth: `assert ':' not in sid` at function entry (mitigates A2, the hypothetical sid-format change).

Module constants `INT_FIELDS = ("input", "output", "cache_read", "cache_write", "total")` and `COST_FIELD = "cost"`. `equal_split` raises `ValueError` if `n < 1`. Both functions assert conservation byte-exact internally (defense-in-depth; same assertions also repeated in T02 + T10 tests).

4-space indentation. LF line endings. Trailing newline. No emojis. Use the canonical algorithm from `03-RESEARCH.md` Pattern 5 (lines 524-583) verbatim for `equal_split`; do not deviate from the remainder-absorption order (last marker absorbs).

Field-name convention (per D-03 Claude's discretion): use the unadorned forms `input, output, cache_read, cache_write, total, cost` (matches existing `hermes-report.sh` local var stems `delta_input/delta_output/delta_cache_read/delta_cache_write/delta_total/delta_cost` with the `delta_` prefix stripped). Thread this convention through `hermes-report.sh` marker-row construction (T04).

**Rationale for `parse_prior_state` extraction (B6):** Without extraction, T10 would either (a) re-implement the field-count discrimination in Python and merely decorate production logic, or (b) shell out to `hermes-report.sh` with a fragile stderr-capture protocol. Extracting the function makes the test load-bearing — a bug in production parsing makes T10 fail — while keeping the function pure and stdlib-only.
