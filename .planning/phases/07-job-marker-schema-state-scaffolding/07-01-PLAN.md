## Trust Boundaries

## STRIDE Threat Register

This phase is pure additive scaffolding: no new network calls, no new
credentials, no new file permissions, no new external input surface. The
`kind:"job"` line is read from a file the agent already writes to, by a reader
that already exists. Per RESEARCH.md Security Domain, no ASVS L1 categories
apply. The register below is minimal by design.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | `kind:"job"` line in `markers/<sid>.jsonl` | mitigate | Malformed / oversized / unknown-`kind` job lines are skipped, never fatal — the reader inherits the existing `try/except json.JSONDecodeError: continue` and 4 KB line cap, and unknown `kind` values hit the `elif kind is not None: continue` forward-compat skip (D-06). A job line missing any JOB_REQUIRED key is dropped, not raised. |
| T-07-02 | Denial of Service | marker reader per-line loop | accept | The reader caps line length at 4096 bytes and processes one line at a time; a hostile job line cannot exceed v1.0's existing per-line cost. No new amplification. Local-only file, single-user host — low-value target. |
| T-07-03 | Information Disclosure | `revenium-jobs.ledger` (touch-created, empty) | accept | File is `touch`-created empty under `~/.hermes/state/revenium/`, same local-only state directory and permissions as every existing ledger. No content written in Phase 7. Same posture as `revenium-hermes.ledger`. |
</threat_model>
