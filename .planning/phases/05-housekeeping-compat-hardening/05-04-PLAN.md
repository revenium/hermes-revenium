## Marker file pruning

`prune-markers.sh` is an operator-invoked script that removes marker files for sessions whose latest reported ledger row is older than the retention window (default 30 days). It is NOT auto-run from cron (D-28) — operators run it periodically when marker files accumulate. The script reads `~/.hermes/state/revenium/revenium-hermes.ledger` to derive the latest-reported timestamp per session id; for orphan marker files with no ledger row, it falls back to file mtime. Stale files are removed and active sessions' marker files are preserved. The script is idempotent across repeated runs.

Usage:

```bash
# Preview which files would be removed without touching them
bash ~/.hermes/skills/revenium/scripts/prune-markers.sh --dry-run

# Actually remove stale files
bash ~/.hermes/skills/revenium/scripts/prune-markers.sh
```

Both invocations log per-file actions to `~/.hermes/state/revenium/revenium-metering.log` via the standard `info` log helper and emit a final summary line (`scanned=<n> kept=<n> removed=<n>`).

To change the retention window from the default 30 days, export `REVENIUM_MARKER_RETENTION_DAYS` before invoking:

```bash
REVENIUM_MARKER_RETENTION_DAYS=60 bash ~/.hermes/skills/revenium/scripts/prune-markers.sh
```
