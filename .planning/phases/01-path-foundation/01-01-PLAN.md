<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local user → `~/.hermes/state/revenium/markers` | Marker JSONL files (Phase 2+) will carry task_type strings; if other local users on the same host can read them, task labels leak |
| Env vars → install paths | `HERMES_HOME` and `REVENIUM_MARKERS_DIR` env overrides redirect where state lives; a hostile env can point at a victim path before `chmod 700` runs |
| Test assertions → `common.sh` text | A future contributor could satisfy the path-discipline test by adding the literal as a comment instead of a real variable declaration, weakening the invariant silently |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information disclosure | `~/.hermes/state/revenium/markers/` (multi-user host) | mitigate | PATH-02: `chmod 700 "${MARKERS_DIR}"` in `install-cron.sh` after `common.sh` is sourced; idempotent on re-run; verified by `stat -f %A` / `stat -c %a` returning `700` |
| T-01-02 | Tampering / privilege confusion | `MARKERS_DIR` env override redirecting to attacker-controlled path | accept | Same trust boundary as `HERMES_HOME` — an attacker who controls env vars can already redirect every existing state path. `chmod 700` narrows permissions on the env-resolved target, which is the safe direction (does not expand attacker access). Severity: low. Documented as a known property of the env-override design. |
| T-01-03 | Tampering (invariant bypass) | `test_runtime_paths_are_hermes_native` substring-only check | mitigate | PATH-03: in addition to substring assertions (`task-taxonomy.json`, `markers`), assert variable-assignment patterns (`TAXONOMY_FILE=`, regex for `MARKERS_DIR=` with `:-` env-fallback). Comment-only or string-in-helper satisfaction will fail the test. |
| T-01-04 | Denial of service | `mkdir -p "${MARKERS_DIR}"` on every cron tick adds disk I/O | accept | `mkdir -p` on an existing directory is a near-no-op syscall pair (stat + return). The directory is created on first source and persists; subsequent sources are O(1). Severity: trivial. |
| T-01-05 | Repudiation | Changes to chmod state are not logged | accept | `install-cron.sh` is operator-invoked, not cron-invoked. Operator sees `chmod 700` execute (or fail under `set -euo pipefail`). No silent state change. Severity: trivial. |

Block on: high severity. No high threats in this phase. T-01-01 (medium) is mitigated by PATH-02. T-01-03 (low-medium) is mitigated by the tightened test assertion in Task 3.
