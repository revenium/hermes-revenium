<objective>
Add `state.db.messages` lookup to the classifier so the LLM receives the actual session content (last user message + last assistant message) instead of empty strings.

Purpose: The plugin entrypoint `__init__.py:_on_session_end` always passes `message=None, response=None` to `run_classification`. Today that means `_classify_via_llm` builds the mint-first prompt with empty `user_preview` and `asst_preview` strings. Even after the 260514-nfb mint-first rewrite, the LLM has nothing to classify from — it falls back to bland labels like `generation` / `unknown_task`. Reading the last user + last assistant messages from `state.db.messages` gives the LLM the substance it needs to mint specific labels (`weekly_pr_review`, `prod_log_triage`, etc.).

Output:
- ONE new module-level helper `_read_session_messages(sid)` in `classifier.py`.
- ONE modified block (Step 5 area inside `run_classification_async`) that consults the helper only when caller passed None.
- ONE new test `test_revenium_classifier_reads_state_db_content` in `tests/test_repository.py`.
- All 41 tests green after T01; SUMMARY.md committed in T02.
</objective>
